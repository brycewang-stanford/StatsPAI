# R package versions backing the reference-parity fixtures

The frozen reference values in `tests/reference_parity/_fixtures/*_R.json` were
generated by the `_generate_*.R` scripts against the **specific R package
versions recorded below**. A parity test compares StatsPAI to these frozen
numbers, so if an upstream R package changes its algorithm, the frozen fixture
can quietly become stale while the test still passes against the old number.
This manifest makes that risk visible: when regenerating fixtures, record the
versions you used here, and if they differ from the row below, re-verify that
the new R output still matches the old before overwriting the fixture.

## Regenerating a fixture

```bash
Rscript tests/reference_parity/_fixtures/_generate_<name>.R
```

After regenerating:

1. Confirm the produced numbers match the committed `*_R.json` within the
   documented tolerance (see `REFERENCES.md`). If they do **not**, an upstream R
   package changed — investigate before overwriting, and note the version bump
   here.
2. Update the version row above if you used a newer R / package set.
3. A fixture change that moves a StatsPAI parity number is a numerical change —
   treat it under the correctness-fix process in
   [`docs/guides/reproducibility.md`](../../docs/guides/reproducibility.md).
