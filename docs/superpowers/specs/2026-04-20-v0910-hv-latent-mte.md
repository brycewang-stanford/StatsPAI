# StatsPAI v0.9.10 — HV-latent MTE (true Heckman-Vytlacil via latent U_D)

**Author:** Bryce Wang · **Date:** 2026-04-20 · **Status:** design → implementation

## 1. Motivation

v0.9.9 shipped `sp.bayes_mte(..., first_stage='plugin' | 'joint')` and explicitly labelled the fitted curve as **treatment-effect-at-propensity** `g(p) = E[Y|D=1,P=p] - E[Y|D=0,P=p]` rather than the textbook MTE `tau(u) = E[Y_1 - Y_0 | U_D = u]`. Under Heckman-Vytlacil (2005) linear-separable outcome with bivariate-normal errors, `g(p) = tau(p)`; under arbitrary heterogeneity the two diverge.

The honesty fix was the right call for v0.9.9. v0.9.10 closes the debt by adding the **actual HV-latent model**: sample `U_D_i ~ Uniform(0,1)` per unit, conditioned on observed `D_i` and posterior propensity `p_i`, then evaluate the MTE polynomial at each unit's latent `U_D_i`.

## 2. Scope

### In scope (v0.9.10)

- **`sp.bayes_mte(..., mte_method='polynomial' | 'hv_latent')`** — new kwarg orthogonal to the existing `first_stage` kwarg.
  - `'polynomial'` (default): v0.9.9 behaviour — polynomial in `p_i`, i.e. `g(p)`.
  - `'hv_latent'`: polynomial in `U_D_i` where `U_D_i` is sampled per unit from the HV-correct truncated uniform.
- **Implementation** — use the reparameterisation trick so NUTS stays applicable:
  ```
  raw_U_i ~ Uniform(0, 1)   (shape=n)
  U_D_i = raw_U_i * p_i                 if D_i = 1   (U_D < p iff D=1)
        = p_i + raw_U_i * (1 - p_i)     if D_i = 0
  tau(u) = b_0 + b_1 u + ... + b_{poly_u} u^{poly_u}
  Y_i = alpha + beta_X' X_i + D_i * tau(U_D_i) + eps_i
  ```
  This avoids `pm.Truncated` and keeps NUTS happy.
- **Method label** — `hv_latent` returns with method label `"Bayesian HV-latent MTE (poly_u=..., ... first stage)"`; `polynomial` keeps the v0.9.9 label `"Bayesian treatment-effect-at-propensity"`.

### Orthogonality contract

All four (first_stage, mte_method) combinations run and are documented:

| first_stage \ mte_method | `polynomial` | `hv_latent` |
|---|---|---|
| `plugin` | v0.9.8 — fixed p_i, polynomial in p | **NEW** — fixed p_i, latent U_D_i |
| `joint`  | v0.9.9 — Bayes p_i, polynomial in p | **NEW** — Bayes p_i, latent U_D_i (most honest, slowest) |

### Out of scope (deferred)

- Full bivariate-normal error structure on `(U_0, U_1, U_D)` — current model is linear-separable with no selection-on-levels. Natural 0.9.11+ extension.
- Multi-instrument MTE.
- GP over `u` (still polynomial).
- Rust Phase 2.

## 3. API

```python
def bayes_mte(
    data, y, treat, instrument, covariates=None,
    *,
    first_stage: str = 'plugin',     # 'plugin' | 'joint'
    mte_method: str = 'polynomial',  # 'polynomial' | 'hv_latent'
    u_grid=None,
    poly_u: int = 2,
    ...
)
```

Validation: reject any `mte_method` outside the whitelist with a clear error at call time.

## 4. Test plan

- `test_bayes_mte_hv_latent_runs` — API smoke.
- `test_bayes_mte_hv_latent_recovers_true_mte_on_hv_dgp` — DGP where outcome = `alpha + MTE(U_D_i) * D_i + eps`; `hv_latent` should recover the polynomial coefficients; `polynomial` mode should NOT.
- `test_bayes_mte_hv_latent_plus_joint_runs` — orthogonality with `first_stage='joint'`.
- `test_bayes_mte_invalid_mte_method_raises`.
- `test_bayes_mte_method_label_reflects_hv_latent`.
- `test_bayes_mte_hv_latent_method_info` — records `mte_method='hv_latent'`.
- Retain all v0.9.9 `polynomial`-mode tests unchanged.

## 5. Success criteria

1. On a DGP where the outcome equation is `alpha + tau(U_D_i) * D_i + eps` (genuine HV), `mte_method='hv_latent'` recovers the true MTE polynomial within HDI at n=800, while `mte_method='polynomial'` has a systematic bias (non-zero diff between the two posterior means).
2. All four (first_stage, mte_method) combos run without error on a small DGP.
3. Full suite stays green vs v0.9.9 baseline (2054 passed).
4. Round B + Round C code reviews pass with zero blockers.

## 6. Non-goals

- Prior-tuning for the latent U_D (we use `raw_U_i ~ Uniform(0,1)`, which is the correct uninformative prior given the HV assumption of uniform latent U_D).
- Full bivariate-normal HV model (that adds correlated error structure between Y_0 and Y_1 given U_D).
- Warnings on large n (the latent U_D adds `n` parameters to the NUTS state; we document the slow-down without auto-capping).
